Contributing to SYSeg
===================================

Thank you for your interest in SYSeg!

This project is open source, and we welcome contributions from the community
to help improve SYSeg.

How to Contribute
-------------------------------------

If you ever:

* find a technical problem in the source code
* notice a conceptual inaccuracy in the examples
* want to suggest a new feature
* improve code examples or documentation

your contribution will be very welcome.

To proceed, please

* read [docs/syseg.md](docs/syseg.md)
* open an issue in the repository
* optionally, contact the author

See the [AUTHORS](AUTHORS) file for contact details.

Licensing
------------------------------------

The project adopts the following licensing convention:

* source code, build scripts, and related infrastructure files use `GPL-3.0-or-later`
* `README` files and the main manual use `CC-BY-SA-4.0`
* other documentation files, such as `AUTHORS`, use `CC-BY-4.0`
* auxiliary low-expression files, such as `.gitignore`, use `CC0-1.0`

Code convention
------------------------------------

This project follows these conventions:

* Semantic Versioning: <https://semver.org/spec/v2.0.0.html>

* Keep a Changelog: <https://keepachangelog.com/en/1.0.0/>

* Symbol names, comments, file names, documentation, and similar content are 
  in English.

* Source code style follows the conventions of each programming language.

* For C code, and with a heavy heart, we humbly apologize to K&R for opening
  braces on the next line for both branches and functions. (I know, I know...)
